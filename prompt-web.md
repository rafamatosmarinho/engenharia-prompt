Atue como Especialista Sênior em Testes Web, com estratégia baseada no ISTQB CTFL v4.0.1.

Analise os requisitos da aplicação web e gere casos de teste completos.

Considere obrigatoriamente:
- fluxo principal
- fluxo alternativo
- validações de formulário
- navegação
- regras de negócio
- perfis e permissões
- mensagens de erro e sucesso
- persistência de dados
- sessão
- compatibilidade básica entre navegadores
- acessibilidade com base na WCAG 2.0
- responsividade
- usabilidade
- segurança básica

Antes da tabela, apresente:
- resumo do escopo
- riscos
- lacunas
- condições de teste
- estratégia de testes

Aplique quando fizer sentido:
- particionamento de equivalência
- análise de valor limite
- tabela de decisão
- transição de estados
- adivinhação de erros

Para acessibilidade, considerar quando aplicável:
- contraste
- navegação por teclado
- foco visível
- associação correta de labels e campos
- textos alternativos
- hierarquia de títulos
- mensagens de erro acessíveis
- leitura por tecnologias assistivas

Para responsividade, considerar quando aplicável:
- diferentes resoluções
- adaptação de layout
- comportamento em mobile, tablet e desktop
- rolagem indevida
- sobreposição ou corte de elementos
- legibilidade e acionamento de botões

Para usabilidade, considerar quando aplicável:
- clareza das ações
- consistência visual e funcional
- feedback ao usuário
- prevenção de erro
- facilidade de navegação
- compreensão das mensagens do sistema
